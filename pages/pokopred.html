Why it exists

Reliable betting intelligence without bespoke plumbing

PokoPred targets bettors and analysts who need consistent, explainable predictions but don’t want to wire up ingestion, storage, or modeling stacks from scratch. The system emphasizes reliability, transparency, and a guided user experience.

  • Automated season ingestion keeps raw files synchronized and verifiable.
  • Explainable signals highlight feature contributions and draw confidence miles ahead of kick-off.
  • Guided UX shields less technical users from brittle scripts while staying automation-friendly.

CLI rails · cli_pokopred_documentation.md

Interactive shell + scriptable switches

The CLI opens with a banner-driven menu that steps through setup, season updates, model training, and specialist analysis tools. Every menu choice double checks intent, streams progress states, and pauses at milestones so new operators stay oriented.

  • clipokopred --setup|--update|--train|--analysis mirrors each menu branch for schedulers.
  • Confirmation gates prevent destructive actions on production datasets.
  • Progress + pause screens make long model runs feel like a guided walkthrough.

Data spine · data_betproj_postgres_documentation.md

Resilient PostgreSQL pipeline

Automated orchestration stretches from ingestion to inference so every prediction is backed by auditable lineage.

  • DataDL synchronizes historical + live seasons, feeding DbPostgresRawDataStore.
  • DbTeamStatsStore computes advanced form metrics; PostgresModelStore builds the feature matrix.
  • ModelRun schedules training + inference, while DrawDataProcessor tracks dedicated draw probabilities.

Harvest layer · data_dler_documentation.md

Respectful scraping + fixture intelligence

DataManager manages polite sessions, league-aware file naming, and CSV validation. UpComingGames builds the forward-looking fixture dataset so predictions stay actionable.

  • Utility helpers track which leagues map to each season (England, Germany, Spain, Italy, France, more).
  • Harvest checkpoints surface schema drift before it hits downstream stores.
  • Operators can extend coverage by dropping new league definitions instead of rewriting code.

Model stack · data_models_documentation.md

Consistent training interfaces and ensembles

All models inherit the PokoPred base class so loading, preprocessing, evaluation, and persistence behave identically whether you deploy Random Forests or Neural Nets.

  • Bundled models: Random Forest, Logistic Regression, Neural Network, XGBoost, plus calibrated + soft voting ensembles.
  • EnsembleDrawAnalyzer merges outputs into ROI-centric summaries, not just accuracy charts.
  • DrawDataProcessor and draw-specific models keep conservative bettors informed.

Draw conversion + observability

Auditable prediction upgrades and logging

draw_conversion_implementation.md details how low-confidence home/away picks flip to draws when dedicated thresholds are crossed. StoreEnhancedPredictions writes enriched results back to PostgreSQL with conflict-safe upserts for full traceability.

  • logging_config_documentation.md centralizes JSON configs, module overrides, and rotating handlers.
  • Unified logging makes debugging harvesting jobs, long model runs, or CLI workflows tractable on remote servers.
  • Structured metadata lets monitoring dashboards plot accuracy + ROI drifts per release.

Tooling + roadmap

Analysis suite and living documentation

The supporting tools catalog (tools_documentation.md) keeps analysts inside the guided shell while exploring ROI, accuracy, draw strategies, and weather overlays.

  • models_prediction_analysis.py, hyperparameters_optimizer.py, and weather/map loaders plug directly into the CLI.
  • CHANGELOG.md, docs/changelog_patches, and version.json document releases (2.0.0 merged runs, centralized logging, new analysis flows).
