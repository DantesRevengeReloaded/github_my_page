PokoScribe

PokoScribe is a Multi-Agent System (MAS) that combines summarization, citation, paper outlining, and chapter creation for automation of academic writing.

PokoScribe Repo

Explore the PokoScribe repository ↗

The system has agents that work together to speed up academic writing tasks or can be used individually for specific tasks if the data are provided in the correct format.

Prompts and answers of the system are stored using PostgreSQL database.

Important Notice

PokoScribe is not a replacement for human writers and under no circumstances should the output of the system be used as the final version of the paper. The system is designed to help writers to write faster and more efficiently automating some of the tasks in the writing process and only if the data are gathered with rationality and have been prepared in the correct format.

Agents

PokoScribe leans on three specialists that trade work in sequence. Each agent writes to the same schema so the next one can pull context without brittle parsing.

Start

Summarizer

Breaks long-form papers into batches, keeps citations attached, and stores token counts for downstream limits.

  • Splits large PDFs into safe chunks with consistent IDs.
  • Captures citation strings alongside every summary paragraph.
  • Persists prompts/answers in PostgreSQL for reuse.

Structure

Outliner

Reads the summaries table and drafts scaffolds with references mapped back to the source material.

  • Pulls only the summaries for the active project ID.
  • Suggests headings plus inline citations per section.
  • Writes outline JSON that the next agent can stream.

Draft

Chapter Creator

Expands the outline into prose, keeping chunk sizes API-safe and reusing the stored citations.

  • Consumes outline nodes and stitches their source refs.
  • Enforces token budgets per batch before generation.
  • Publishes chapters back to the database for review.

Acquisition layer

Research capture stays citation-ready

The collection toolkit keeps every PDF and metadata fragment aligned with the project ID so agents can stitch sources without manual cleanup.

  • AHSS sweeps discovery engines and persists structured references for later citation injection.
  • SciHub-dler mirrors the DOI map, naming files the same way the database expects.
  • TokenCounter tags every chunk with token counts so prompts stay inside the guardrails.

Agent relay

Specialists hand off work like a studio

Summaries, outlines, and chapters are treated as linked deliverables: each agent writes to the same schema so the next one can ingest without brittle parsing.

  • Summarizer batches long-form sources, saving citations + token metadata alongside the copy.
  • Outliner queries the summaries table, generating section scaffolds with references to upstream sources.
  • Chapter Creator reuses those references and enforces API-friendly chunk sizes before publishing prose.

Quality loop

Human review stays in the circuit

Automation speeds drafting, but the workflow still routes every deliverable through an editorial pass so ethics and rigor remain intact.

  • Prepare prompts with explicit tone + citation demands before triggering a full run.
  • Spot-check generated copy against the stored references to confirm fidelity.
  • Merge agents or run them solo depending on whether you need a quick excerpt or an end-to-end draft.
